How it works

1 - Choose a Tweet

Have you spotted a suspicious tweet today? Or do you simply want to find out how a tweet spreads within the social network? Then, copy the URL of that tweet, paste it into the search bar and hit the search button.

2 - Gather its traces

The TraceMap starts to unfold from the user who first shared the post. TraceMap's open-source algorithms display up-to-date information, showing how that tweet spread through the social network's users.

3 - Explore the map

TraceMaps are easy to interpret: each node represents a user and their sizes indicate how influential they are. Click on a node to find out more about it. Connections between users reveal how information flows. Trace all possible paths!

4 - Share your findings

What did you discover? Share the TraceMap and your findings with your friends and followers. Get their feedback and start a crowd-debunking process.

In Depth Information

This section explains the data and algorithms we use to generate TraceMaps in detail.

1. Our Database

We use a Neo4J graph database to store Information about Twitter users.
Each user is saved as a node in this graph. Each link is directed and tells, that user-a is following user-b. We receive this data through the API of Twitter.

2. The tweet data

If a user submits a Tweet Url, we request the Twitter API live. We receive information about the last 100 retweets including the time and user id of each retweet.

3. Database Request

Now, we request our database, receiving the sub-network of followship between all the retweeters of the original tweet.

4. Merging the data

Finally, we merge the information from Twitter and our Database, to generate the TraceMap.
Therefore we first connect all the retweeters following the author of the tweet. Then, we connect each of the remaining nodes, if the user follows one of the already connected ones.
The link is just added if the source retweet took time before the target one.

Without the database, the resulting network would always have the author directly connected to each retweeting user.
Our logic enables us to guess, which followers of a retweeter brought the information to their timeline.
Of course its still a guess and sometimes there is more than one possibility of a connection. But each TraceMap shows some unique metadata from which we can extract useful information.

Our Next Steps

Of course this is just one part of the TraceMap project.
From now on, we focus on analyzing the TraceMaps of certain Tweet categories e.g. Fake News, Hatespeech, Propaganda or just informative Tweets.
If they show a difference in propagating, the resulting TraceMaps will also differ. After manual tagging, we could use machine learning to assess the category of each Tweet.
